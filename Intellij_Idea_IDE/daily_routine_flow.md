## 🔁 Daily Routine (do this every coding session)

### 1. 🔥 Warm-up (2 mins)

**force yourself to use shortcuts only**:
* ⇧ ⇧ → search a class
* ⌥ ⌘ ← / → → move back & forward
* ⇧ ⌃ R → run something

### 2. ✍️ Editing drill (5 mins)

**repeat every day**:
* ⌘ N → generate constructor
* ⌥ ⏎ → fix/import something
* ⌘ / → comment/uncomment
* ⌘ D → duplicate lines
* ⇧ ⌥ ↑/↓ → move lines
* ⌘ ⌫ → delete lines
* ⌥ ⌘ R (rename) → rename variable
* ⌘ B → go to declaration

### 3. 🧼 Cleanup habit (2 mins)

* ⌃ ⌥ O → optimize imports
* ⌥ ⌘ L → reformat code
**automatic at the end of every change**

### No Mouse
* ⌘ 1 ↑/↓ → select project panel

### 4. 🧠 “One new shortcut focus” (2–3 mins)

Pick **1 shortcut per day** and overuse it:

Example:
* Day 1 → ⌥ ⏎ (use it constantly)
* Day 2 → ⌘ N
* Day 3 → ⌘ ⌥ ← / →

### 5. 🚫 Constraint rule (all day)

Force this rule while coding:

> “If I know a shortcut exists → I must use it”

No mouse fallback.

---

## ⚡ Memory hack (important)

Group them mentally:

* **Navigation** → ⇧⇧, ⌥⌘← →
* **Editing** → ⌘D, ⌘⌫, ⇧⌥
* **Fixing** → ⌥⏎
* **Cleanup** → ⌃⌥O, ⌥⌘L
* **Run** → ⇧⌃R

👉 Brain remembers **clusters, not lists**
