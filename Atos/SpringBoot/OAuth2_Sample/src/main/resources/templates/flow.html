End-to-end flow

Authorization Code Flow

What happens when you click “Login with OAuth2” in this sample app.

  1. Authorization request

    The client redirects the browser to /oauth2/authorize with client_id, redirect_uri, response_type=code, and scope.

  2. User authentication

    If not signed in, Spring Security shows the login page. Demo users authenticate with username/password.

  3. Consent

    The authorization server asks the user to approve the requested scopes for client demo-client.

  4. Authorization code

    The browser is redirected back to /login/oauth2/code/demo-client?code=... with a one-time authorization code.

  5. Token exchange

    The OAuth2 client exchanges the code at /oauth2/token using client_id and client_secret, receiving access and refresh tokens.

  6. Resource access

    The access token (JWT) is sent as Authorization: Bearer ... to protected endpoints such as /api/user.

Key endpoints

Endpoint Purpose
/.well-known/openid-configuration OIDC discovery document
/oauth2/authorize Start authorization code flow
/oauth2/token Exchange code or refresh token for tokens
/oauth2/jwks Public keys for JWT validation
/userinfo OIDC user info (requires access token)
/api/public/health Unprotected resource server endpoint
/api/user Protected resource server endpoint
