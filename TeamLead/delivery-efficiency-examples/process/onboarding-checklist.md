# Engineer Onboarding Checklist

Reduces cognitive load — complete in first 2 weeks.

## Day 1

- [ ] Laptop + SSO + GitHub org access
- [ ] `devbox shell` in any service repo — local stack up
- [ ] Backstage: bookmark team services and runbooks
- [ ] Slack: #team-channel, #platform-support, on-call alias (read-only)

## Week 1

- [ ] Read golden paths: create-service, deploy-app, kafka-integration
- [ ] Ship small PR (docs or test) through full CI
- [ ] Pair on incident drill or shadow on-call
- [ ] Try internal RAG: ask 3 questions from `knowledge-sharing/rag/sample-questions-and-answers.md`

## Week 2

- [ ] Own a ticket end-to-end using AI first-draft workflow
- [ ] Review someone else's PR using AI summary + human checklist
- [ ] Optional: add one prompt improvement to `ai-practices/prompt-library/`
